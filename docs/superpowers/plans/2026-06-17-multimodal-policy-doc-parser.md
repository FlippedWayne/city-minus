# Multimodal Policy Document Parser Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add optional multimodal PDF image/chart parsing using PyMuPDF extraction plus Volcengine Ark VLM descriptions, producing `DocumentChunk(chunk_type="image")` only when the VLM description passes a quality gate.

**Architecture:** Keep text parsing unchanged by default. When `MULTIMODAL_PARSE_ENABLED=1`, `DocumentParser._parse_pdf()` appends image chunks generated by `multimodal_parser.extract_image_chunks()`. `vision_client.py` wraps Ark VLM via the existing OpenAI SDK, with image-description caching under `data/cache/vision`.

**Tech Stack:** PyMuPDF, OpenAI-compatible Volcengine Ark VLM API, existing Pydantic-free config dataclasses, pytest.
